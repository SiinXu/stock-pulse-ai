# 10 Settings (UI operations)

Path: **Settings**.  
After each group of changes, click **Save** and wait for success before leaving the page.

## AI and models

1. Add a model provider (Anspire, AIHubMix, OpenAI-compatible, local, and others as listed).
2. Paste the key and select or fetch models.
3. Set the primary analysis model; optionally set a dedicated chat/Agent model.
4. Use **Test connection**.
5. For local models, use the local-model panel to browse, install/register, then select (when the client exposes that panel).

## Basics and watchlist

- Edit the watchlist; separate codes with commas.
- After save, Home and Analysis watchlist entry points update.

## Data sources and news

- Fill enhanced market or search keys as needed.
- Basic technical analysis may still run without news keys, but event quality drops.

## Notifications

- Fill webhook, bot token, email, and related fields per channel.
- Use on-page test send when available.
- Adjust detail level and per-stock notify behavior using the switches shown in UI.

## Other common items

- UI language and report language
- Theme
- Admin password when login is enabled
- Usage and cost dashboard
- Config backup import/export (especially useful on desktop)

## Out of scope here

Account signup on key vendors, server ports, and Docker mounts belong in deployment docs, not this UI manual.
